# Model experiments — bake-off against the shipped models

> Generated by `scripts/experiment_models.py`. Changes nothing that ships (no model overwrite, no metrics.json write). A winner is promoted separately and deliberately.

## Engine 2 — next-technique prediction

Test = 780 prediction points across 35 held-out sequences (vocab 566, OOV next-techniques counted as misses: 45). Only 140 training sequences — small-data regime.

| Method | top-1 | top-3 | top-5 | vs shipped (top-3) |
|---|---|---|---|---|
| Most-frequent (baseline) | 1.9% | 4.9% | 8.6% | -31.7 pts |
| Kill-chain order (baseline) | 3.6% | 7.1% | 9.2% | -29.5 pts |
| Markov 1st-order (SHIPPED) | 22.8% | 36.5% | 44.5% |  |
| Markov 2nd-order (new) | 21.4% | 33.2% | 38.6% | -3.3 pts |
| Markov interpolated l=(0.2, 0.3, 0.5) (new) | 23.1% | 38.1% | 44.4% | +1.5 pts **<-- best** |
| LSTM (existing) | 14.4% | 27.2% | 38.2% | -9.4 pts |
| biLSTM mean-pool (new) | 10.6% | 20.0% | 28.5% | -16.5 pts |

- Anti-circularity still holds for the shipped model: Markov 1st-order top-3 (36.5%) is 5.2x the kill-chain baseline (7.1%).
- **A variant beats the shipped model: Markov interpolated l=(0.2, 0.3, 0.5) (new) at 38.1% top-3 (+1.5 pts).** Promotion needs a runtime change (the API serves a 1st-order table); do it deliberately, not from this script.
- **Significance:** paired bootstrap over the 780 test prediction points, 96% of 2,000 resamples keep Markov interpolated l=(0.2, 0.3, 0.5) (new) ahead of the shipped model. That clears a 95% bar, so the gain is real.
- The neural variants both lose. The biLSTM does worse than the plain LSTM: mean-pooling the prefix dilutes the most recent technique, which is exactly the signal that predicts the next one, and it carries twice the parameters on 140 training sequences.
